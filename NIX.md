# Nix Environment — llmtelemetry

Reference for maintaining, updating, and reverting the Nix shell for this project.

## Files

| File | Purpose |
|------|---------|
| `default.R` | Source of truth — edit this to add/remove packages, then regenerate |
| `default.nix` | Generated by `rix::rix()` via `default.R`; manually patched for `otelsdk` |
| `default.post.sh` | Idempotent post-regeneration script; re-applies the `otelsdk` derivation block |

**Never edit `default.nix` directly** unless adding/updating the `otelsdk` block.
All other changes go through `default.R` → regenerate → `default.post.sh`.

## Entering the shell

```bash
nix-shell /Users/johngavin/docs_gh/llmtelemetry/default.nix
```

From inside another nix-shell (agent subshells, CI):

```bash
nix-shell /Users/johngavin/docs_gh/llmtelemetry/default.nix --run "Rscript -e '...'"
```

`--pure` is not required; the `shellHook` (if present) corrects `R_LIBS_SITE` contamination.

## Adding or removing R packages

1. Edit `default.R` — add to `r_pkgs` or `system_pkgs`
2. Regenerate `default.nix`:
   ```bash
   nix-shell ~/docs_gh/llm/default.nix --run \
     "(cd /Users/johngavin/docs_gh/llmtelemetry && Rscript default.R)"
   ```
3. Re-apply the `otelsdk` block (idempotent):
   ```bash
   /Users/johngavin/docs_gh/llmtelemetry/default.post.sh
   ```
4. Verify:
   ```bash
   nix-shell /Users/johngavin/docs_gh/llmtelemetry/default.nix \
     --run "Rscript -e 'library(otelsdk); library(otel); cat(\"OK\n\")'"
   ```

## The otelsdk derivation

`otelsdk` is GitHub-only (not on CRAN). `rix::rix()` cannot add `nativeBuildInputs`,
so the derivation is written manually and lives between the `rpkgs`/`system_packages`
blocks and `shell = pkgs.mkShell {`.

**Why each piece exists:**

| Piece | Why |
|-------|-----|
| `pkgs.which` in `nativeBuildInputs` | configure script calls `which cmake`; `which` is not in the Nix derivation sandbox PATH |
| `pkgs.cmake` in `nativeBuildInputs` | obvious; also sets the cmake binary path |
| `pkgs.protobuf_21` in `nativeBuildInputs` | otelsdk requires protobuf 3.21.x; protobuf_21 has no Abseil dependency, compatible with `-DWITH_ABSEIL=OFF` |
| `pkgs.curl`, `pkgs.curl.dev` in `nativeBuildInputs` | cmake MODULE-mode `FindCURL` needs both the lib and the headers |
| `pkgs.zlib`, `pkgs.zlib.dev` in `nativeBuildInputs` | same for `FindZLIB` |
| `preBuild: export CMAKE=...` | configure reads `$CMAKE`; avoids `which cmake` lookup which fails in Nix sandbox |
| `CMAKE_PREFIX_PATH` **NOT** set | macOS system `/usr/lib/libcurl.dylib` is accessible in the Nix Darwin sandbox; semicolons in `CMAKE_PREFIX_PATH` become bash statement separators inside Makefile recipes → "Is a directory" errors |
| `postPatch: sed on src/Makevars.in` | `src/CMakeLists.txt` is inside `src/vendor/opentelemetry-cpp.tgz` (extracted at build time, not patchable at patch time); `src/Makevars.in` IS patchable and controls all cmake flags |
| `-DFETCHCONTENT_FULLY_DISCONNECTED=ON` in `postPatch` | `cmake/googletest.cmake` uses `FetchContent_MakeAvailable(googletest)` which git-clones at configure time — blocked in Nix sandbox; cmake 4.x evaluates `FetchContent` before `BUILD_TESTING` guards take effect |

### Pinned revision

```
rev = "0cf2ad93631944d28142e941ae9ac5cab107f478"  # otelsdk 0.2.4.9000 (2026-05-11)
```

To update to a new release:
1. Find the new commit SHA from https://github.com/r-lib/otelsdk
2. Update `rev` in `default.nix` AND in `default.post.sh`
3. Update the `sha256` by running:
   ```bash
   nix-prefetch-git https://github.com/r-lib/otelsdk <new-rev>
   ```
4. Run `default.post.sh` (won't re-insert if already present; edit manually if updating rev)
5. Rebuild and verify

## Reverting the Nix environment

### Revert to a previous `default.nix`

```bash
git -C /Users/johngavin/docs_gh/llmtelemetry show <commit>:default.nix > /tmp/default.nix.old
diff /Users/johngavin/docs_gh/llmtelemetry/default.nix /tmp/default.nix.old
# Review diff, then:
git -C /Users/johngavin/docs_gh/llmtelemetry checkout <commit> -- default.nix
```

### Revert otelsdk block only (keep other changes)

```bash
# Remove the otelsdk block and restore buildInputs to [ rpkgs system_packages ]:
git -C /Users/johngavin/docs_gh/llmtelemetry diff main~5 main -- default.nix | head -80
# Then edit default.nix manually to remove the otelsdk = pkgs.rPackages.buildRPackage { ... }; block
# and revert buildInputs = [ rpkgs system_packages otelsdk ] to buildInputs = [ rpkgs system_packages ]
```

### Nuclear: regenerate from scratch

```bash
nix-shell ~/docs_gh/llm/default.nix --run \
  "(cd /Users/johngavin/docs_gh/llmtelemetry && Rscript default.R)"
# This overwrites default.nix completely from default.R — no otelsdk block.
# Re-apply only if otelsdk is wanted:
/Users/johngavin/docs_gh/llmtelemetry/default.post.sh
```

## Nixpkgs date pin

The environment is pinned to `2026-01-05` (rstats-on-nix fork):

```nix
pkgs = import (fetchTarball "https://github.com/rstats-on-nix/nixpkgs/archive/2026-01-05.tar.gz") {};
```

Updating the date pin changes ALL package versions simultaneously. Do this deliberately,
not as part of an unrelated change. After updating, rebuild and run the full test suite.

## Known issues / workarounds

| Issue | Workaround |
|-------|-----------|
| `which: command not found` during otelsdk configure | `pkgs.which` in `nativeBuildInputs` (fixed 2026-05-12) |
| googletest git-clone blocked in Nix sandbox | `-DFETCHCONTENT_FULLY_DISCONNECTED=ON` via `src/Makevars.in` patch (fixed 2026-05-12) |
| `CMAKE_PREFIX_PATH` semicolons break Makefile | Do NOT set `CMAKE_PREFIX_PATH`; macOS system paths work in Nix Darwin sandbox (fixed 2026-05-12) |
| R_LIBS_SITE contamination when nesting nix-shells | Add `shellHook` that rebuilds `R_LIBS_SITE` from closure (see global `nix-nested-shell-isolation` rule) |
